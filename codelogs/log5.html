Images in HTML

How are images used in HTML and CSS?

Images can be added to a webpage in two main ways. In HTML, you use the <img> element, such as <img src="example.jpg">, which places an actual image into the pages content. In CSS, you can apply an image as a background using background-image: url("example.jpg");. Both methods display images, however theyre used in different situations depending on what the image is meant to do, whether that be decorative or be there for content.

Whats the difference of what method you choose?

HTML and CSS can both show images, but they work differently. HTML images are part of the pages content and have their own natural size. CSS background images are decorative and only appear if the element they are applied to has size. Background images can repeat, cover, or be positioned in ways that HTML images cant. In short, HTML is used for content images, while CSS is used for decorative backgrounds.

What CSS properties affect background images?

Besides background-image, there are several CSS properties that change how a background image appears on a website. One of the most important is background-size, which controls how large the image is, such as using cover or contain. Another useful property is background-position, which lets you choose where the image sits inside the element, like centering it or aligning it to a certain corner.

What is the difference between dimensions and resolution?

Dimensions describe the pixel width and height of an image, while resolution describes how densely those pixels are packed when the image is displayed or printed.

What are the similarities and differences between GIF, PNG, JPG/JPEG, webP, and SVG?

GIF, PNG, JPG/JPEG, WebP, and SVG are all image formats, but they work differently and are better for different uses. GIF supports simple animations and only uses 256 colors, making it good for basic graphics. PNG uses lossless compression and supports full transparency, which is ideal for logos or images that need sharp edges. JPG/JPEG uses lossy compression, so it creates samll file sizes but is best for photos rather than graphics. WebP is a newer format that can be either lossy or lossless, offering smaller file sizes than PNG or JPG while keeping good quality. SVG is completely different because it is a vector format, meaning it can scale to any size without losing quality, making it ideal for things like icons.
