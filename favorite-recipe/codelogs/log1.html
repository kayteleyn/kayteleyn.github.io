HTML Semantic Tags

What is the difference between non-semantic and semantic tags in HTML?

The differences between non-semantic and semantic tags in HTML are purely aesthetic and functionality. Semantic tags tell the browser what is actually going on with the code, such as <header> and <section>, in which both tell the browser how to format the page. With non-semantic, they dont have a meaning, such as <b>, which just bolds the text, being only something visual rather than something that needs to be formatted.

What are two or more benefits of using semantic tags?

Semantic tags are useful for countless things in HTML. Semantic tags are a hige resource when building a web page, it formats the photos, paragraphs, ordered and unordered lists, and so many other cool functions. Such as <ol>, it stands for ordered list, and creates bullets for everything within the list, making it easy to follow along to a tutorial, or just reading off of a list, obviously.So essentially, semantic tags are both helpful for easy reading, and for organization.
